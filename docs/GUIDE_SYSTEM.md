# Guide System

## Goal

Make `solfege` useful even before full music generation is perfect.

The Guide should be a real product module, not filler.

It should help users:

- learn DJing and mixing basics
- understand music theory in a practical way
- improve sound design and sound engineering
- get project-aware advice for the song they are building
- save notes and learning threads

## Recommended Guide UI

The Guide should be its own main tab.

Inside that tab, use 3 areas:

### 1. Guide Chat

An AI tutor chat where users can ask questions like:

- "How do I blend these two tracks?"
- "What intro fits this project?"
- "How do I mix a distorted vocal?"
- "How do I create more tension at 145 BPM?"

Important features:

- conversation history
- project-aware context
- quick follow-up prompts
- "Create note" action on any answer

### 2. Topics Library

A browsable list of structured learning topics.

This should include:

- built-in lessons
- saved custom topics
- recently viewed topics
- recommended next topics

### 3. Notes

A dedicated notes area where users can save:

- AI answers
- project-specific takeaways
- workflow reminders
- personal techniques
- favorite settings or rules

This turns the Guide into a real knowledge workspace.

## Core Guide Sections

## A. Music Fundamentals

This section should cover:

- beats and tempo
- notes and scales
- chords
- basslines
- melodies
- song structure

This is strongly supported by Ableton's Learning Music resources.

## B. DJ Fundamentals

This section should cover:

- beatmatching
- sync and when to use it
- harmonic mixing
- Camelot wheel basics
- phrasing and transitions
- energy flow

This should stay practical and not overcomplicate theory.

## C. Sound Engineering

This section should cover:

- EQ basics
- compression basics
- saturation and distortion
- loudness basics
- balancing kick and bass
- controlling harshness and mud
- arrangement for clarity

This is where many users will get real long-term value.

## D. Sound Design

This section should cover:

- oscillators
- waveforms
- filters
- envelopes
- modulation
- sampling and slicing
- turning reference ideas into playable sounds

## E. Project-Aware Guidance

This is the most special part.

The Guide should understand the user's current project and answer things like:

- what should the intro do?
- what layer is missing?
- is this idea too busy?
- what kind of vocal fits this BPM and mood?
- how should I transition from breakdown to drop?

This is where DeepSeek can become very useful.

## Guide Content Model

Each topic should have:

- title
- category
- difficulty
- short summary
- key ideas
- examples
- practical actions
- related topics

Each saved note should have:

- note title
- note body
- source chat or topic
- optional linked project
- tags

## AI Role In The Guide

DeepSeek or another text model can power:

- question answering
- lesson summaries
- personalized explanations
- project-aware advice
- note generation
- quiz or exercise generation later

The AI should not be the only source of truth.

The best experience is:

- curated guide topics first
- AI adapts and explains them
- users save and organize what matters

## Research-Backed Curriculum Seed

These sources are strong starting points for the guide system:

### Ableton

Ableton's Learning Music covers:

- beats
- tempo
- scales
- chords
- basslines
- melodies
- song structure

Ableton's educational resources also point users to Learning Synths and broader Live learning.

### Mixed In Key

Mixed In Key's harmonic mixing guides are useful for:

- Camelot wheel basics
- compatible key moves
- same-key and adjacent-key transitions
- major/minor switching for harmonic blends

### Serato

Serato support material is useful for:

- beatmatching displays
- sync modes
- beatgrids
- understanding that sync helps timing, but does not auto-mix a real set

### iZotope

iZotope educational material is useful for:

- EQ basics
- compression basics
- mix bus compression
- loudness concepts
- mastering mindset

## Product Rule For The Guide

The Guide should teach practical decisions, not just definitions.

Bad:

- "Compression reduces dynamic range."

Better:

- "Your kick is too jumpy and your bass is getting lost. Try a slower attack on the compressor and re-check how the low-end breathes."

## Good First Guide Topics

For version 1, start with:

- Beatmatching and sync
- Harmonic mixing and Camelot basics
- Phrasing and transitions
- BPM and energy flow
- EQ for beginners
- Compression for beginners
- Loudness basics
- Intro / breakdown / drop structure
- How to write a better hook
- How to choose references intelligently

## V1 Guide Features

- built-in topic cards
- AI tutor chat

- project-aware questions
- note history
- search topics and notes

## Later Guide Features

- quizzes

- project checklists
