# Open Questions

These are the biggest decisions still worth discussing before serious coding starts.

## Product Scope

- Should version 1 produce only text and project outputs, or also short audio?
- Should the app focus first on underground electronic users, or stay broad from the start?
- How strong should the learning and guidance layer be in the first release?

## Reference Handling

- Will users paste links only, or upload files too?
- Will the app fetch media from links directly, or only use the links as references?
- How should we handle copyrighted music and public-figure meme material?

## AI Strategy

- Which model should power the text brain first?
- Which provider or approach should later power audio generation?
- Do we want one provider at first, or an adapter layer from the start?

## Product Experience

- What should the main workspace look like?
- Should the 2-3 references appear as separate panels, cards, or timeline inputs?
- What should the user see first after analysis finishes?

## Business

- Is the first monetization path subscriptions, credits, or premium exports?
- Will there be a free tier?
- How expensive can generation be before the model breaks financially?

## Technical

- Web-only first, or plan a desktop client early?
- Clerk or Auth.js?
- Prisma or Drizzle?
- Supabase storage or S3-compatible storage?
